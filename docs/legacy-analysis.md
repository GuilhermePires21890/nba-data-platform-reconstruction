# Legacy Project Analysis

## Original Context

The original project was developed as a university assignment focused on historical NBA player statistics extraction and analysis.

The solution combined web scraping, CSV exports, SQL processing and statistical analysis.

## Original Technologies

| Area | Technology |
|---|---|
| Web Scraping | Selenium |
| Main Language | C# |
| Data Storage | CSV |
| Database | SQL Server |
| Statistical Analysis | R |
| Visualization | ggplot2 |

## Legacy Pipeline

1. NBA statistics website
2. Selenium scraper
3. CSV generation
4. SQL transformations
5. Statistical analysis in R

## Positive Aspects

- End-to-end data pipeline thinking
- Historical dataset organization
- Multi-technology integration
- Analytical mindset
- Statistical exploration

## Technical Limitations

| Area | Limitation |
|---|---|
| Storage | CSV fragmentation by season |
| SQL | Repetitive scripts |
| Pipeline | Manual execution |
| Architecture | Monolithic structure |
| Reproducibility | Environment dependency |
| Automation | Limited orchestration |

## Reconstruction Goals

The modernization process aims to:

- modernize the architecture
- improve maintainability
